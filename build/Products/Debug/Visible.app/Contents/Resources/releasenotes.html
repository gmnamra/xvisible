Visible Release Notes

  

Visible 1.0

10/2003

System Requirements

  • Disable computer and hard drive sleep from System Preferences. Video capture process will fail if the system goes to sleep. Display sleep does not have to be disabled.
  • Do NOT run any other user applications during video capture.
  • Video analysis requires large amounts of memory so 2 GB of RAM is recommended for best performance.

Licensing

  • License file format has been changed and Visible 0.92 license files will not work any more. A new license file will have to be generated by Reify.

Limitations

  • Visible 1.0 is an early beta version with limited functionality.
  • Visible 1.0 video capture duration is limited to 524288 frames. All captures in Reify format can be analyzed without limits. However, when movies in Quicktime format are analyzed, Visible will attempt to load all video frames into memory for analysis and OS X virtual memory size is limited to 3 GB. If video frame size is 1280x960, the maximum analyzable movie length is 1000-2000 frames depending on analysis operation.
  • If motion graphics are produced during Functional visual analysis, they may consume large amounts of memory depending on input data. Each motion vector graphic token consumes 20 bytes so a video frame containing a vector field of 100 000 vectors would consume 2 MB of memory...thus no more than 1500 such frames can be analyzed without exhausting all virtual memory.
  • Reify Experiment Markup Language (.rfyeml) is at a preliminary stage and it is subject to change. It cannot be guaranteed that future product versions can read old .rfyeml result files. However, results can be saved in comma-separated-value CSV format for easy Excel-import.
  • Reify Experiment Markup Language (.rfyeml) result files contain the path name of the analyzed movie. If the movie cannot be found in its original location, Visible will attempt to find it in the same directory where .rfyeml file is located.

New Features

  • Added new analysis operation Functional visual analysis.
  • Added new analysis operation Muscle cell analysis.
  • Added new entropy definition settings for entropy analysis operations:
    1. ACI
    2. Visual Entropy
    3. Diffusion Index
  • Added settings to choose first and last video frame to analyze; subsets of frames within the movie can now be analyzed.
  • Added View menu in menu bar. It contains two items:
    1. Locomotive Body Graphical Data Display
    2. Locomotive Body Text Data Display
  • Added new item Open Experiment Template in File menu.
  • Added new item New Experiment in File menu.
  • Added new settings tab Visualize which contains movie playback controls and settings for functional visual analysis data visualization.
  • Added new settings tab Export which contains settings to control result data export.
  • Added new "rectangle zoom" mode in video monitor.
  • ACI sliding window graph is now displayed during capture preview and analysis.
  • ACI sliding window settings can now be changed during capture preview.
  • Added new settings under Experiment settings tab: imaging mode, temperature, CO2 level, O2 level, treatment2.
  • Experiment settings are now stored in .rfymov files instead of .rfyeml files.
  • Added new capture information text display areas under Capture settings tab: camera information, capture data information, capture conversion information.

Bug Fixes and Enhancements

  • Changed some experiment settings from text fields to menu choices: cell/organism type, lens magnification, other magnification.
  • Improved stability.
  • Improved video and graphics display speed.

Known Issues

  • On rare occasions a few video frames may be lost at the beginning of video capture. If this happens, an error dialog about dropped frames will be displayed when the capture has finished. This does not corrupt captured video or measurement data of the session, it just means that some frames may have a longer than usual frame duration.

Visible 0.92

02/2003

System Requirements

  • Disable computer and hard drive sleep from System Preferences. Video capture process will fail if the system goes to sleep. Display sleep does not have to be disabled.
  • Do NOT run any other user applications during video capture.
  • Video analysis requires large amounts of memory so 2 GB of RAM is recommended for best performance.

Limitations

  • Visible 0.92 is an early alpha version with limited functionality.
  • Visible product license is bound to OS X host ID which is generated from primary IP address. Thus if the IP address of the machine changes, the license will not work and a new license must be generated.
  • Visible 0.92 video capture duration is limited only by available disk space. However, during analysis Visible will attempt to load all video frames into memory for analysis and OS X virtual memory size is limited to 3 GB. If video frame size is 1280x960, the maximum analyzable movie length is 1000-2000 frames.
  • Reify Experiment Markup Language (.rfyeml) is at a preliminary stage and it is subject to change. It cannot be guaranteed that future product versions can read old .rfyeml result files. However, results can be saved in comma-separated-value CSV format for easy Excel-import.
  • Reify Experiment Markup Language (.rfyeml) result files contain the absolute path name of the analyzed movie. Thus if the captured movie is renamed or moved after analysis results have been saved, loading the result file into Visible will fail because the movie cannot be located.

Known Issues

  • On rare occasions a few video frames may be lost at the beginning of video capture. If this happens, an error dialog about dropped frames will be displayed when the capture has finished. This does not corrupt captured video or measurement data of the session, it just means that some frames may have a longer than usual frame duration.
