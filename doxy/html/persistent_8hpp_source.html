286 // Supports null pointers too!
287 
288 // Warning! The pointer must be a dynamically-allocated type, since the implementation uses new/delete
289 // If the data field to restore is null and the file format non-null, allocates a new T()
290 // If the data field is non-null and the file format is null, deletes it and sets it null
291 
292 // Multiple pointers to the same object *will* be restored as multiple
293 // pointers to the same object. The object is dumped only the first time it is
294 // encountered along with a "magic key". Subsequent pointers to the same
295 // object cause only the magic key to be dumped. On restore, the object is
296 // only restored once and the magic keys are matched up so that the other
297 // pointers now pojnt to the restored object.
