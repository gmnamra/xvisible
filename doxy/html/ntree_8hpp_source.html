130 // Traversal iterators are like iterators but they have increment operators (++)
131 // - prefix_iterator visits the nodes of the tree in prefix order
132 // - postfix_iterator visits the nodes of the tree in postfix order.
133 // There is no such thing as infix order for an n-ary tree and you cannot
134 // traverse backwards with these iterators. These follow the STL convention in
135 // that you iterate from a begin to an end - in this case ntree exports
136 // prefix_begin()/prefix_end() and postfix_begin()/postfix_end(). You can
137 // simplify these iterators to the basic iterator above for functions that
138 // require a simple iterator.
