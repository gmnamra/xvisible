45 // Read-only versus read-write control. This is equivalent to chmod on Unix,
46 // but I've insulated the user from the low-level routine because of
47 // differences in the OSs' interpretation of the mode parameter. I've also
48 // defined a new set of constants to control this, again because of
49 // inconsistencies. The idea is to combine the constants as bit-masks so as to
50 // build up a set of permissions. The modes are ORed together to build up a
51 // set of permissions and then ANDed with a mask to control which people have
52 // that permission. Permissions can be ORed together too. So, for example, to
53 // give the owner read-write access and all others only read access, you would
54 // use the expression:
55 // ((read_mode | write_mode) & owner_mask) | (read_mode & (group_mask | other_mask))
56 // This can be simplified by using combined modes and combined masks to:
57 // (read_write_mode & owner_mask) | (read_mode & non_owner_mask)
