73  /* rcVideoCacheCtor - Opens file and initializes data structures. If
74  * getTOC is set to true, then the table of contents (a map of
75  * timestamps to images) is read in.
76  *
77  * Note: maxMemory puts a limit on the number of frames that will be
78  * maintained in cache by specifying the maximum number of bytes
79  * of memory the cached frames can take up. Specifying a 0 means
80  * there is no limit.
81  *
82  * cacheSize specifies the number of frame buffers to maintain
83  * in cache. Specifying a value of 0 means there is no limit.
84  *
85  * The actual number of cache slots to maintain is the minimum of
86  * the following triplet: floor(maxMemory/frameSize), (iff != 0)
87  * cacheSize, (iff != 0)
88  * frameCount
89  *
90  * The previous discussion uses the term "maintain in cache" not
91  * "maximum cache size" because the implementation will attempt
92  * to allocate entries when the cache size is exceeded. When an
93  * overflow is about to occur, the cache first removes any
94  * unlocked frames from the cache. When these run out, it then
95  * allocates additional frames from the heap. While in an
96  * oveflow state, any newly unlocked frames are immediately
97  * removed from the cache. This continues until the number of
98  * frame buffers in cache is within the limits specified to the
99  * ctor.
100  *
101  */

103  uint32 cacheSize, bool getTOC,
