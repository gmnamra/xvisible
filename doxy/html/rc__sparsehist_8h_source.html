9 /* rcSparseHistogram - Allows users to create a "bounded" sparse
10  * histogram. By bounded is meant that the maximum number of bins to
11  * be used can be specified in at object construction time. The intent
12  * was to provide a reasonably fast, memory efficient, way of
13  * generating a sparse histogram. This was done using STL maps and
14  * this class is mostly a thin wrapper around map functionality.
15  *
16  * Note: If a way could be devised to guarantee that all the memory
17  * required will be allocated a construction time this would be a
18  * plus. This class is being used during real time operations and
19  * anything that can prevent the need to allocate heap would be a
20  * help. STL maps don't seem to provide a way of directly specifying
21  * how many elements you will be using. Maybe elements could be
22  * preallocated by creating a map that has the requested number of
23  * elements, using keys that are out of range. When a new bin is
24  * needed one of these preallocated entries could be taken and its key
25  * modified. Whether or not maps would allow this sort of scheme needs
26  * to be investigated.
27  */
