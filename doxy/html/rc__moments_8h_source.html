278  /* gen3by3Point - Perform fast autocorrelation for the "model" at
279  * the center of the given search space and return the result in
280  * res. Correlation score will be generated using integral images
281  * created by call to update.
282  *
283  * See rsCorr3by3Point for a description of the result structure.
284  * The size of the model the operation is being performed upon
285  * always has dimensions srchSpace.width()-2 by
286  * srchSpace.height()-2.
287  *
288  * Note: if the object was created with fast == true, then the model
289  * size must be <= 256.
290  *
291  * The number of correlations performed is determined by the
292  * arguments passed to update(). See description of update()
293  * for details.
294  *
295  * minVarTimesSz is a measure of variance. To be exact, it is
296  * VARIANCE*N*(N-1), where N is the number of pixels in the
297  * model. This choice was made to avoid the cost of performing the
298  * division required to calculate the true variance.
299  *
300  * Since the correlation of a model with itself is always 1, the
301  * calculated value of the variance measure for the model is stored
302  * at the center location of the result structure (score[1][1]).
303  * When this value is less than minVarTimesSz, none of the
304  * correlation scores are calculated and the values stored in the
305  * rest of the result structure are all undefined. Setting
306  * minVarTimeSz to -1.0 guarantees that the correlation values are
307  * always returned.
308  *
309  * Note: update() must be called at least once before this fct gets
310  * called. The search space must refer to an rcRect that fits within
311  * the last rcWindow passed to update() and be at minimum, 3 by 3.
312  */
313  void gen3by3Point(const rcRect& srchSpace, rsCorr3by3Point& res, resultSpace space = eFull) const;

332  /* genPoint - Perform autocorrelation for the "model" at the center
333  * of the given search space and return the result in res. Does not
334  * depend upon integral images to generate results, but will use any
335  * existing integral images to speed up processing.
336  *
337  * The result is a 2D vector of correlation scores. The inner/outer
338  * indices specify vertical/horizontal locations in the image
339  * coordinate system, respectively. In other words, assuming a 5 by
340  * 5 result was requested, res[0][4] is the far upper-far right
341  * correlation score array and res[2][0] is the middle-far left
342  * correlation score array.
343  *
344  * acRadius determines the size of the resulting autocorrelation
345  * space. It will be N x N where N = (1 + acRadius*2). res
346  * must be at least large enough to hold this.
